## 페이지 흐름 요약

```
[Nav] → [S1 Hero] → [S2 숫자 임팩트] → [S3 왜 미쳤다고 불리는가?]
     → [S4 120분 프로그램] → [S5 참가 팀] → [S6 우승팀 혜택]
     → [S7 참여 신청 CTA] → [S8 FAQ] → [S9 Final CTA + Footer]
```

### PASONA 매핑 총괄

| PASONA | 섹션 | 역할 |
|--------|------|------|
| **P** (Problem/호기심) | S1 Hero + S3 | 호기심 유발, 왜 와야 하는지 |
| **A** (Agitation/공감) | S2 숫자 + S3 | 규모감으로 신뢰, 문제의식 공감 |
| **S** (Solution) | S4 프로그램 + S5 팀 | 구체적 해결책(이벤트 내용) |
| **O** (Offer) | S6 혜택 | 우승팀에게 돌아가는 가치 |
| **N** (Narrowing) | S7 CTA 영역 | 한정성/긴급성 ("선착순", "마감 임박") |
| **A** (Action) | S7 + S9 | 전환 유도 CTA 2회 반복 |

- 3열 균등 분배, 모바일에서는 1열 세로 스택

- 모바일: 이미지 상단, 텍스트 하단으로 스택
- **개선 제안**: 공감 요소 보강 - "AI 시대, 직접 눈으로 확인하지 않으면 뒤처집니다" 같은 공감 카피 추가 검토

### 카피

| 요소 | 텍스트 | 비고 |
|------|--------|------|
| 섹션 타이틀 | `🏆 참가 팀` | 확정 |

- 4-5열 그리드, 각 카드에 팀 로고 + 팀명

- 각 카드 hover 시 간단한 팀 설명 툴팁 또는 모달
- 모바일: 2열 그리드

### 카피

- 2컬럼 카드 레이아웃: 관객(좌) / 발표자(우)
- 발표자 카드는 **다크 배경 + 초록 CTA 버튼**으로 더 강한 시각적 강조
- 관객 카드는 **밝은 배경 + 테두리**로 심플하게
- **N(한정성) 강화**: "선착순 100명", "마감 D-11" 같은 요소 추가 권장
- 모바일: 1열 세로 스택 (관객 → 발표자 순서)
